Class Reference Documentation

Complete guide to all available classes and their usage

Class Name Category Description Use Case
.bento-grid Container Main grid container with auto-fit columns Required wrapper for all bento layouts
.bento-container Container Max-width container with padding Optional outer wrapper for centered layouts
.bento-item Item Individual grid card/tile Required for each content card in the grid
.featured Item Large featured card (2×2 grid span) Highlight main content or carousel items
.span-2 Modifier Spans 2 columns horizontally Wide cards for longer titles/content
.span-3 Modifier Spans 3 columns horizontally Extra-wide featured content
.row-2 Modifier Spans 2 rows vertically Tall cards with more content space
.row-3 Modifier Spans 3 rows vertically Extra-tall cards for extensive content
.layout-pinterest Layout Masonry-style grid with varying heights Image galleries, portfolios, visual content
.layout-header-banner Layout Large banner top + 6 smaller cards below Landing pages, announcements, hero sections
.layout-balanced Layout Symmetrical 4-column layout with featured item Balanced content presentation, dashboards
.layout-asymmetrical Layout 5-column layout with 3×2 main item Dynamic layouts, editorial content
.tall Pinterest Doubles card height (spans 2 rows) Use within .layout-pinterest for taller cards
.wide Pinterest Doubles card width (spans 2 columns) Use within .layout-pinterest for wider cards
.banner-item Banner Full-width banner at top of layout Use within .layout-header-banner for hero
.content-1 to .content-6 Banner Positioned cards in header banner layout Use within .layout-header-banner for cards
.large-item Balanced 2×2 featured card in balanced layout Use within .layout-balanced for main feature
.medium-1 to .medium-4 Balanced Medium-sized cards in balanced layout Use within .layout-balanced for secondary content
.small-1 to .small-4 Balanced Small cards in balanced layout Use within .layout-balanced for tertiary content
.main-item Asymmetric 3×2 main featured card Use within .layout-asymmetrical for hero
.side-1, .side-2 Asymmetric Sidebar cards in asymmetrical layout Use within .layout-asymmetrical for side content
.article-tag Component Category/tag badge for cards Label content categories
.read-time Component Reading time indicator (bottom-right) Display estimated reading duration
.carousel-container Carousel Wrapper for carousel slides Required container for carousel functionality
.carousel-slide Carousel Individual slide within carousel Each content slide in the carousel
.carousel-image Carousel Image within carousel slide Display images in carousel slides
.carousel-content Carousel Text content wrapper in carousel Contains title and description
.carousel-dots Carousel Dot navigation container Auto-generated by JS for navigation
.dot Carousel Individual navigation dot Auto-generated for each slide
.active Carousel Currently visible slide/dot Managed automatically by framework JS
.bento-title Utility Large page title styling Main headings above grid sections
.skeleton Utility Loading skeleton animation Placeholder while content loads

Usage Tips

  • Always start with .bento-grid as your container
  • Each card must have the .bento-item class
  • Combine modifiers like span-2 row-2 for custom sizes
  • Layout classes go on the .bento-grid, not individual items
  • Carousel requires the .featured class on the item
  • All layouts are fully responsive by default

Customization Guide

Override CSS variables and utility classes to customize the framework to your brand

CSS Variables (Custom Properties)

The framework uses CSS variables defined in :root. Override these in your custom CSS to change colors, spacing, and more:

Variable Default Value Description Affects
--bento-bg-primary #0a0a0a Main background color Body background
--bento-bg-secondary #1a1a1a Card background color All .bento-item backgrounds
--bento-bg-tertiary #2a2a2a Accent background color Tags, dots, hover states
--bento-text-primary #ffffff Primary text color Headings, titles
--bento-text-secondary #e0e0e0 Secondary text color Body text, descriptions
--bento-text-tertiary #888 Tertiary text color Tags, metadata, read-time
--bento-accent #3a3a3a Accent color for interactions Hover borders, focus states
--bento-border #2a2a2a Border color All card borders
--bento-radius 10px Border radius for cards All .bento-item corners
--bento-transition all 0.3s ease Default transition timing Hover effects, animations

How to Override Variables

Add this CSS to your custom stylesheet (after importing bento-grid.css):

Custom CSS Example 
/* Your custom styles.css */
:root {
    /* Change to blue theme */
    --bento-bg-primary: #0a0e1a;
    --bento-bg-secondary: #1a1f2e;
    --bento-bg-tertiary: #2a3142;
    --bento-accent: #3a5aff;
    --bento-border: #2a3142;
    
    /* Rounder corners */
    --bento-radius: 20px;
    
    /* Lighter text */
    --bento-text-tertiary: #a0a0a0;
    
    /* Faster transitions */
    --bento-transition: all 0.2s ease;
}
Light Theme Example 
/* Light theme override */
:root {
    --bento-bg-primary: #f5f5f5;
    --bento-bg-secondary: #ffffff;
    --bento-bg-tertiary: #f0f0f0;
    --bento-text-primary: #1a1a1a;
    --bento-text-secondary: #333333;
    --bento-text-tertiary: #666666;
    --bento-accent: #e0e0e0;
    --bento-border: #e0e0e0;
}

Add Utility Classes to Your CSS

Copy and paste these utility classes into your custom stylesheet:

Utility Classes CSS 
/* Utility Classes - Add to your custom CSS */

/* Background Utilities */
.bg-gradient { background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); }
.bg-dark { background: #0a0a0a; }
.bg-light { background: #2a2a2a; }

/* Border Utilities */
.border-accent { border: 2px solid var(--bento-accent); }
.border-none { border: none; }
.rounded-none { border-radius: 0; }
.rounded-lg { border-radius: 20px; }

/* Padding Utilities */
.p-small { padding: 20px; }
.p-large { padding: 40px; }

/* Shadow Utilities */
.shadow-sm { box-shadow: 0 2px 8px rgba(0,0,0,0.2); }
.shadow-lg { box-shadow: 0 20px 60px rgba(0,0,0,0.5); }
.shadow-none { box-shadow: none; }

/* Hover Utilities */
.hover-lift:hover { transform: translateY(-8px) !important; }
.hover-scale:hover { transform: scale(1.02) !important; }
.no-hover { transform: none !important; pointer-events: none; cursor: default; }
.no-hover:hover { transform: none !important; box-shadow: none !important; }

/* Text Utilities */
.text-center { text-align: center; }
.text-right { text-align: right; }

/* Opacity Utilities */
.opacity-50 { opacity: 0.5; }
.opacity-75 { opacity: 0.75; }

Usage Example

Combine utility classes with bento classes for custom styling:

        <span class="article-tag">Premium</span>
        <h3>Featured Content</h3>
        <p>This card stands out with custom styling.</p>
    </div>

    <!-- Card with no border and extra padding -->
    <div class="bento-item border-none p-large">
        <span class="article-tag">Design</span>
        <h3>Spacious Layout</h3>
        <p>More room for content.</p>
    </div>

    <!-- Centered text card with custom hover -->
    <div class="bento-item text-center hover-scale rounded-lg">
        <span class="article-tag">Special</span>
        <h3>Centered Content</h3>
        <p>Everything is centered with scale hover.</p>
    </div>

    <!-- Disabled/inactive card -->
    <div class="bento-item no-hover opacity-50">
        <span class="article-tag">Coming Soon</span>
        <h3>Inactive Card</h3>
        <p>This content is not yet available.</p>
    </div>
</div>

Customization Best Practices

  • Override CSS variables in a separate custom.css file loaded after the framework
  • Test color contrast for accessibility (WCAG 2.1 AA standards)
  • Keep border-radius consistent across your design system
  • Use utility classes sparingly - create custom classes for repeated patterns
  • Document your custom variables and utilities for team collaboration
  • Preview changes across all layouts before deploying
